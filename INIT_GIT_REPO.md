# Инициализация и перенос проекта в GitHub

Данный документ описывает шаги для инициализации локального репозитория Git и переноса проекта в удаленный репозиторий на GitHub.

## Подготовка проекта

Проект уже подготовлен к публикации:
- Создан файл `.gitignore` для исключения чувствительных и ненужных файлов
- Удалены все файлы с чувствительными данными (`.env`, `frontend/.env`)
- Создан файл `.env.example` с шаблоном переменных окружения
- Обновлен файл `README.md` с описанием проекта

## Инициализация локального репозитория

1. Откройте терминал в корневой директории проекта
2. Выполните команды:

```bash
# Инициализация репозитория
git init

# Добавление всех файлов
git add .

# Создание первого коммита
git commit -m "Initial commit: AI Agent with Meta-Cognition system"
```

## Создание репозитория на GitHub

1. Зайдите на [github.com](https://github.com)
2. Нажмите кнопку "New repository"
3. Укажите имя репозитория (например, `ai-agent-meta-cognitive`)
4. Выберите "Public" для бесплатного размещения
5. Не ставьте галочку "Initialize this repository with a README"
6. Нажмите "Create repository"

## Связывание локального и удаленного репозитория

1. После создания репозитория на GitHub скопируйте HTTPS URL (или SSH URL, если вы настроили SSH-ключи)

2. В терминале выполните команды:

```bash
# Связывание с удаленным репозиторием
git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git

# Установка основной ветки
git branch -M main

# Первый push в удаленный репозиторий
git push -u origin main
```

Замените "ВАШ_ЛОГИН" и "НАЗВАНИЕ_РЕПОЗИТОРИЯ" на соответствующие значения.

## Проверка успешного переноса

После выполнения этих шагов проверьте:
- Все файлы успешно загружены на GitHub
- В репозитории отсутствуют чувствительные данные
- Файлы конфигурации и документации присутствуют

## Настройка развертывания

После переноса проекта в GitHub автоматически активируются настроенные ранее:
- GitHub Actions для CI/CD
- Конфигурация для развертывания на Render.com
- Конфигурация для развертывания фронтенда на GitHub Pages

Для завершения настройки развертывания следуйте инструкциям в файле [README_DEPLOY.md](README_DEPLOY.md).

## Важные замечания

- Не храните чувствительные данные (пароли, API-ключи) в репозитории
- Используйте файлы `.env` только локально, они исключены из репозитория
- Все необходимые переменные окружения указаны в файле `.env.example`
- Для работы приложения в продакшене потребуется настроить переменные окружения в хостинге
